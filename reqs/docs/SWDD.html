2.0 Software Requirements

2.1 Sub Systems

Sub Systems are core components of the smart home core and cannot be enabled/disabled

2.1.1 Message API

2.1.1.1 API shall have a method to send messages SWDD001

The message API shall verify the passed in message object conforms to formatting requirements as defined in messageAPI-ICD.

The message API shall construct a valid message as defined in messageAPI-ICD.

The message API shall construct a valid message as defined in messageAPI-ICD.

2.1.1.2 API shall have a method to get messages SWDD002

The message API shall verify the received message conforms to formatting requirements as defined in messageAPI-ICD.

The message API shall verify the received message is valid as defined in messageAPI-ICD.

The message API shall verify the received message destination matches the current location.

2.1.1.3 API shall have a method to initialize subsystem SWDD003

The message API shall put the lora transceiver into RX Continuous mode after initialization

2.1.1.4 API shall have a method to manage key SWDD004

API shall have a method to update message key as defined in messageAPI-ICD

2.1.2 Lora API

2.1.2.1 API shall have a method to initialize subsystem SWDD005

The lora API shall not allow any use unless lora_port_init() is has been run first.

2.1.2.2 API shall have a method to initialize the lora transceiver to receive continuous data as defined in LoraAPI-Ref-Manual SWDD006

2.1.2.3 API shall have a method to initialize the lora transceiver to transmit data as defined in LoraAPI-Ref-Manual SWDD007

2.1.2.4 API shall have a method for transmitting a payload as defined in LoraAPI-Ref-Manual SWDD008

NOTE: TXâ€™ing of data can only occur after the Lora transceiver has been initialized into TX mode

The lora API shall wait until the transceiver reports finishing transmitting as defined in LoraAPI-Ref-Manual before returning.

The lora API shall return true if a valid payload has been received.

The lora API shall report if errors are present in received data

2.1.3 Console API

2.1.3.1 The console API shall interface over USB/UART SWDD009

2.1.3.2 The console API shall handle the following commands SWDD010

  1. Providing asserts Since Startup

  2. Clearing console screen

  3. Listing available commands to user

2.1.3.3 The console API shall be multi-thread safe SWDD011
